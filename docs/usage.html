Qua / Using Qua

Use the Web REPL

The simplest way to get started is to use the online REPL. Just click on the link to experience Lisp in cyberspace. (Note that this currently requires a keyboard, so it doesn't work on most phones.)

Use Qua in an HTML Page

The next simple way, suitable for small projects, is to create an HTML page and include the file qua.js to evaluate Lisp code.

<!DOCTYPE html>
<html>
<head>
<script type='text/javascript' src='https://qua-lang.github.io/qua-vm/build/out/qua.js'></script>
</head>
<body>
<script type='text/javascript'>
qua.vm().eval_string('($alert "Hello world!")');
</script>
</body>
</html>

You can also put your Qua code into a <script> tag, see the source of this demo page.

Use the Command-Line REPL

For the authentic '60s teletype experience, Qua can also be run from Unix-like command-lines. This requires that you have Node.js installed.

$ npm install -g qua-vm
$ qua
Welcome to Qua!
Press ENTER to evaluate the current line in Qua.
>

The following sections depend on having installed Qua with Node.

Run Qua Code in a File

$ qua-run path/to/file.lisp

Use Qua from Node Programs

Qua is available on NPM as package qua-vm.

var vm = require('qua-vm').vm()
vm.eval_string('(print "Hello world!")')

Create a Bundle for Web Distribution

A bundle is a JS file that includes both a Qua VM as well as your Lisp code (in a preparsed form) for easy distribution. This is currently at the proof-of-concept stage, but is intended to become the main way of distributing Qua-based web apps. There's a demo repository that shows how it works, the synopsis is:

$ qua-bundle input-file.lisp output-file.js

Then include the output JS file in an HTML page and do the following to execute your Lisp code included in the bundle:

qua.vm().eval_user_bytecode()
